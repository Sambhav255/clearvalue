1. Product Requirements Document (PRD)
1.1 Overview
ClearValue is an AI-powered TCO (Total Cost of Ownership) and Business Case Generator built to help enterprise sales and value consulting teams translate complex infrastructure decisions into defensible financial narratives. A user inputs their current storage environment, and ClearValue outputs a 3-year TCO comparison, value lever breakdown, and an AI-generated executive summary ready for a CFO or CIO audience.
1.2 Problem Statement
Problem 1: Value conversations happen too late
By the time a Value Consultant is pulled into a deal, the customer is already deep in vendor evaluation. The economic case is built reactively rather than proactively. Sales reps need a self-serve tool to run a first-pass value snapshot in early discovery before a VC is even involved.
Problem 2: Benchmark libraries are static and stale
Every new product release makes existing TCO templates outdated. Someone has to manually update them. A tool with configurable, documented assumptions solves this at scale.
Problem 3: Technical-to-financial translation does not scale
Sales engineers speak in IOPS and throughput. CFOs speak in NPV and payback periods. Value consultants sit in between but cannot be in every conversation. A tool that auto-converts technical inputs into financial outputs democratizes that translation across the entire sales org.

Lever 1: Power & Cooling Savings
legacyPowerCost = storageTB × 22.5 × 8760 × kwhRate × 1.4 / 1000
everpurePowerCost = storageTB × 2.5 × 8760 × kwhRate × 1.4 / 1000
powerSavings = legacyPowerCost - everpurePowerCost
Source: Everpure published 10x power efficiency advantage. Legacy avg 22.5W/TB, Everpure avg 2.5W/TB. Cooling multiplier 1.4x = industry standard PUE delta.
Lever 2: Refresh & Migration Avoidance
refreshCostPerCycle = (arrayPricePerTB × storageTB × 0.80) + migrationLaborCost
annualRefreshCost = refreshCostPerCycle / refreshCycleYears
refreshSavings = annualRefreshCost  // Everpure Evergreen = $0
Source: Industry standard forklift estimate. Everpure Evergreen subscription eliminates refresh cost entirely.
Lever 3: Admin Labor Reduction
currentAdminCost = adminFTEs × adminFullyLoadedCost × (adminTimePct / 100)
adminSavings = currentAdminCost × 0.32
Source: ESG Economic Validation of Pure1, 2024. 32% reduction in storage administration labor.
Lever 4: Downtime Risk Mitigation
legacyDowntimeHours = 4.38   // 99.99% availability
everpureDowntimeHours = 0.0526  // 99.9999% availability
downtimeSavings = (4.38 - 0.0526) × downtimeCostPerHour
Source: Availability delta. Legacy 99.99% = 4.38 hrs/yr. Everpure SLA 99.9999% = 0.053 hrs/yr.
Summary Metrics
annualSavings = powerSavings + refreshSavings + adminSavings + downtimeSavings
scenarioMultiplier: conservative=0.60, base=1.00, optimistic=1.25
totalSavings3yr = annualSavings × 3 × scenarioMultiplier
npv = Σ (annualSavings × multiplier / 1.08^year) for years 1-3
paybackMonths = (everpurePremium / annualSavings) × 12
roi = ((totalSavings3yr - everpurePremium) / everpurePremium) × 100

2.3 State Management
Two React contexts only — no Redux or Zustand needed:
• InputContext: stores all UserInputs from Screen 1, exposes updateField() and applyPreset()
• ResultsContext: derives TCOResults from InputContext via useMemo, stores scenario (conservative/base/optimistic)
2.4 Gemini API Call Pattern
const response = await fetch(GEMINI_ENDPOINT, {
  method: 'POST',
  body: JSON.stringify({
    model: 'gemini-1.5-flash',
    contents: [{ parts: [{ text: buildPrompt(params) }] }],
    generationConfig: { temperature: 0.4, maxOutputTokens: 400 }
  })
});

3. User Flow Diagram
(See the visual diagram generated in the conversation. Reference this section for written flow description.)
3.1 Primary Flow (Happy Path)
• User arrives at Screen 0 (Landing) — clicks 'Build Your Value Case'
• Navigates to Screen 1 (Inputs) — clicks preset (e.g. Mid-Market), all fields autofill
• User reviews/overrides any fields — clicks 'Calculate Value'
• Screen 2 (Dashboard) loads — KPI cards, scenario toggle, charts, lever breakdown with tooltips
• User optionally toggles Conservative / Optimistic scenario — numbers update reactively
• User clicks 'Generate Executive Summary'
• Screen 3 (Summary) loads — clicks 'Generate', Gemini returns 4-paragraph memo
• User clicks 'Copy to Clipboard' — done
3.2 Back Navigation
• Screen 2 has '← Edit Inputs' button — returns to Screen 1 with all values preserved
• Screen 3 has '← Back to Dashboard' button — returns to Screen 2
• Screen 3 has 'Start New Analysis' button — resets all state, returns to Screen 0
3.3 Navigation Guard
If a user navigates directly to /dashboard or /summary without having entered inputs (storageTB === 0), React Router redirects them to /inputs. This prevents broken states during demos.
3.4 Error States
• Input validation: 'Calculate Value' button disabled if storageTB is 0. Inline error shown if required fields are empty.
• Gemini API error: fallback message shown, KPI cards still visible. 'Regenerate' button remains active.
• Payback period Infinity (zero savings): display 'N/A' instead of a division error.
